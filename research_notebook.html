Abstract

본 문서는 2CRM 라인의 두께 실측값 예측 모델 개발을 위한 선행 연구 2년치 (2024.01 ~ 2025.11)의 핵심 기록을 요약한 연구노트이다. 2024년 상반기 문제 정의 및 국내외 선행연구 조사를 시작으로, 2024년 하반기 예비 데이터 수집 및 특성 분석, 2025년 상반기 후보 모델군 비교 검토를 거쳐, 2025년 하반기에 집중적으로 산출물을 생성하였다. 최종적으로 2025년 11월에 배치/실시간 이중 모델 구조(ver21 + v24 ONNX)로 모델 아키텍처를 확정하였으며, 본 사업(2025.11.27 착수)의 기술적 기반이 되었다.

I. 문제 정의 및 선행연구 조사2024.01 – 2024.06

1.1 연구 배경 및 문제 정의

DKIS 2CRM 라인은 냉간압연 공정 중 두께 편차가 품질 등급 결정의 핵심 변수이나, 현행 공정 제어는 AGC(Automatic Gauge Control)의 실시간 피드백에 의존한다. 피드백 지연에 따른 선단부 두께 편차가 주요 불량 원인으로 지목되어 왔으며, 센서 데이터 기반의 예측 제어(Predictive Control)로의 전환이 요구되었다.

1.2 연구 목표 설정

  • 1차 목표: 2CRM 통과 중 코일의 두께 실측값(thickness measured)을 1초 이내 예측
  • 2차 목표: 예측값과 설정값의 편차를 바탕으로 공정 이상 조기 탐지
  • 3차 목표: 패스 스케줄(Pass Schedule) 자동 최적화를 위한 보조 모델 제시

1.3 선행연구 조사 결과

국내외 문헌 및 경쟁사 사례 조사를 통해 다음 세 가지 접근법이 지배적임을 확인하였다.

접근법대표 사례강점 / 약점
물리 기반 모델
(Mill Stretch Model)		 POSCO, JFE 등 해석 가능성 높음 / 비선형 외란에 취약
선형 회귀 + 피처 엔지니어링 Nippon Steel 초기 연구 단순·빠름 / 복잡 패턴 학습 불가

FINDING 트리 기반 앙상블(LightGBM)이 철강 공정 수치예측 분야에서 RMSE 기준 선형 회귀 대비 20~40% 개선을 보인 연구가 다수 확인됨. 본 연구의 1차 후보 모델로 선정함이 타당하다고 판단.

2.2 탐색적 데이터 분석 결과

2024-09-08 [EDA LOG]
센서 태그 상관 분석 수행. 상위 약 70개 태그가 두께 변동에 유의미한 상관(|r| > 0.1)을 보임. 특히 DK_2CR_IBA_DB230_REAL60 (총 압연력)과 DB230_REAL4 (실측 두께)가 압도적 상관. 다만 DB230_REAL4는 target 자체일 가능성이 있어 leakage 여부 확인 필요.
2024-10-22 [EDA LOG]
피처 간 다중공선성 문제 확인. CPU2_DB801 시리즈와 IBA_DB230 시리즈 일부가 상관계수 0.99 이상. 단순 drop보다는 모델이 자동 선택하게 두는 것이 나을 수 있음. 트리 기반 모델의 robust한 특성이 여기서 유리.
2024-11-30 [LAB NOTE]
기상 데이터 유의미성 검토. 계절성(월/요일/주말)과 두께 편차 사이에 약한 상관(r ≈ 0.05~0.10) 확인. 단독으로는 약하나, 센서 피처와 결합 시 잔차 설명력 개선을 기대할 수 있음.

2.3 가설 1차 검증 결과

FINDING 2주치 샘플 데이터로 LightGBM 기본 파라미터 예비 학습 결과, RMSE ≈ 15~22μm로 측정됨. 목표(10μm)에 미달하나, 데이터 양 부족 및 피처 정제 부족을 고려하면 가설 자체는 유효한 것으로 판단. 본 실험은 대용량 데이터와 정제된 피처에서 재검증 필요.

2.4 주요 리스크 식별

RISK R-1: Machbase 대용량 조회 시 타임아웃 및 OOM 빈발. 본격 학습 전 저장소 이관 필요성 시사.
R-2: 센서 태그 명세(PLC 태그명 ↔ 물리적 의미)의 서류 기준이 실제 데이터와 불일치할 가능성. 현장 검증 필요.
R-3: 강종별(SC강/일반강/SK강) 편차가 크므로 단일 모델 vs 분기 모델 전략 결정이 필요.

FINDING LightGBM이 속도·정확도·scalability 측면에서 최우수. XGBoost와 접전이나 학습 속도 및 메모리 효율에서 LightGBM이 유리. 1D-CNN은 학습 비용 대비 개선폭 제한적이므로 주 모델 후보에서 제외.

3.3 선형 회귀 모델의 보조적 가치 재확인

RMSE는 높으나, 선형 회귀는 계수의 방향성(+/-)을 제공하여 현장 엔지니어의 해석에 유리하다. 본 연구는 LightGBM(주 예측) + 선형 회귀(해석 보조)의 2종 병용 운영을 권고하는 방향으로 정리.

2025-04-18 [LAB NOTE]
PSM(Pass Schedule Management) 보조 모델 아이디어 제안. 센서 태그 대신 공정 변수(두께/속도/장력)만으로 압연력을 예측하는 도메인 특화 모델. 주 예측 모델과는 피처 세트가 완전히 다르므로 별도 모델로 유지하는 것이 타당. 강종별로 모델을 분기하는 방안도 병행 검토.

3.4 상반기 결론

DECISION 주 모델: LightGBM 확정. 해석 보조로 Ridge 선형 회귀 병용. PSM 도메인 모델은 별도 트랙으로 2025 H2에 실험 진행.

4.2 피처 엔지니어링 프로토콜 확정

  • 센서 태그: DK_2CR_* 패턴만 사용 (2CRM 한정)
  • 시간 피처: YEAR/MONTH/DATE + MORNING/EVENING/NIGHT 3구간 원핫
  • 범주형: RAWSPC_ID, PURCOM_ID, WEEKEND 원핫 인코딩
  • 기상: TEMP, HUMI, WIND, ATM, SUN, SNOW, PRECIPITATION, GROUD_TEMP (8종)
  • 결측치: 센서 상수값 제거, 90% 이상 NaN 태그 drop
  • 단위 정규화: μm / mm 혼재 방지를 위한 사전 스케일링

4.3 실험 1: 단일 모델 vs 강종별 분기 모델

2025-08-14 [EXPERIMENT]
실험 조건: 동일 데이터(3개월 샘플), LightGBM 고정 파라미터.
결과:
  • 단일 모델 RMSE: 14.2μm (전체 평균)
  • 강종별 분기 모델 RMSE: SC강 12.8 / 일반강 13.5 / SK강 17.9 (편향 큼)
해석: SK강 샘플 수 부족으로 분기 모델이 오히려 불안정. 주 예측 모델은 단일 모델 유지, 대신 PSM에서 강종 분기를 맡기는 것으로 역할 분담.

4.4 실험 2: 피처 수 축소 효과

피처 수RMSE (μm)학습 시간모델 크기비고
200+ (전체)14.26.2h31 MBbaseline

FINDING 피처 수 68 → 50으로 축소 시 RMSE는 약 0.4μm 증가하지만, 모델 크기 6배, 학습 시간 30% 감소. 실시간 서비스 요구사항(모델 로딩 < 1초)을 고려하면 50 피처 경량 모델의 가치가 크다.

4.5 리스크 R-2 재확인

RISK 예비 실험 중 DB230_REAL4(실측 두께)와 DB230_REAL108(목표 두께)의 태그 명세가 현장 PLC 문서와 불일치하는 사례 확인. 본 사업 실장착 단계에서 반드시 현장 검증 필요. 본 예비 실험에서는 수동 매핑 테이블을 임시 사용.

V. 모델 확정 및 이중화2025.10 – 2025.11

2025년 10월부터 11월까지는 본 연구의 핵심 산출물이 집중된 시기이다. 누적된 실험 결과와 현장 요구사항을 바탕으로, 최종 모델 구조를 확정하고 배치/실시간 요구사항을 동시에 만족시키기 위한 모델 이중화를 설계하였다.

5.1 모델 요구사항 재정리 (2025.10.05)

2025-10-05 [REQUIREMENTS]

5.3 ONNX 변환 타당성 검증

  • LightGBM 단건 추론: 약 200ms
  • ONNX 단건 추론: 약 8ms (25배 개선)
FINDING ONNX 변환의 정확도 손실 없음 + 속도 20배 이상 개선이 입증됨. 실시간 서비스 탑재에 대한 기술적 장애가 제거됨.
