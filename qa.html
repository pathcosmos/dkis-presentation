1왜 LightGBM을 선택했나요? LSTM이나 Transformer는 검토하지 않았나요?빈출기술
핵심 답변 (30초): LSTM도 검토했으나, 학습 속도, 해석 가능성, 정확도 세 측면에서 LightGBM이 현장에 더 적합했습니다. 실제로 RMSE 0.012mm를 달성하여 목표를 초과했습니다.
상세 답변:

1) 학습 속도: 수천만 건 데이터에서 LSTM은 수 시간, LightGBM은 70초. 현장 서버(GPU 없음)의 자원 제약을 고려하면 실용적 선택입니다.

2) 해석 가능성 (Explainability): Tree 기반 모델은 Feature Importance를 직관적으로 제공합니다. 현장 엔지니어가 "어떤 변수가 품질에 영향을 미치는지" 바로 파악할 수 있습니다. 딥러닝은 블랙박스 성격이 강합니다.

3) 정확도: RMSE 0.012mm로 목표(0.015mm)를 20% 초과 달성. 이 정확도에서 딥러닝의 추가 복잡도를 정당화할 근거가 부족했습니다.

6Director App의 유사코일 매칭 알고리즘을 상세히 설명해주세요.빈출기술
핵심 답변: 5가지 가중 유사도 점수로 ClickHouse에서 유사 코일 Top 5를 실시간 매칭합니다.
유사도 공식:
Score = 0.35×강종 + 0.30×두께 + 0.15×패스수 + 0.10×최근성 + 0.10×품질점수

요소가중치계산 방식
강종 유사도35%동일 강종 = 1.0, 다르면 0.0
두께 유사도30%1 - |목표두께 차이| / 허용오차(0.3mm)
패스 수 유사도15%1 - |패스 수 차이| / 최대 차이
최근성10%exp(-생산일수 / 30) → 최근 코일 우선
품질 점수10%과거 성공률 / 100
운영 파라미터: 최근 90일 내 코일 대상 | Top 5 선택 | 5분간 캐시 | 두께 허용오차 0.3mm

후속 처리:
매칭된 코일의 패스별 파라미터(평균, p10, p90)를 참조값으로 추출 → 현재 값과 Gap 분석 → 신호등(GREEN/YELLOW/RED) → P(개선)/P(유지)/P(악화) 확률 → 작업자 적용/무시 → 30초 후 자동 결과 평가

12Materialized Columns의 원리와 88% 절감은?기술
ClickHouse의 Materialized Columns는 INSERT 시점에 SQL 표현식을 자동 실행하여 파생 컬럼을 생성합니다.

자동 처리 항목:
  • 결측치 처리: ifNull(column, default)
  • 스케일링: 표준화 연산
  • 파생 변수: 압하율, 장력비, 속도 변동률
효과: Python 오버헤드(데이터 로딩, pandas 변환, GC)를 ClickHouse C++ 엔진으로 대체.
결과: ver21(600초) → ver24(70초) = 88% 절감.

29RMSE 0.012mm가 실제 현장에서 의미 있는 수준인가요?압박
답변 전략: 겸손하되 자신감 있게. 수치 자체보다 "활용 가치"에 초점.
네, 의미 있는 수준입니다. 냉간압연의 두께 허용 오차는 일반적으로 ±0.01~0.05mm이며, RMSE 0.012mm는 이 범위 내에서 높은 정확도입니다.

비교: Linear Regression MAE 2.49mm → LightGBM MAE 0.31mm로 87% 개선. 이는 현장에서 "사후 측정" 없이도 품질 이탈을 사전에 감지할 수 있는 수준입니다.

물론 모델 정확도 향상은 지속적으로 추진하며, 특히 PSM(Pass Schedule Management) 최적화와 함께 실질적 품질 개선 효과를 높여 나갈 계획입니다.
30현장 작업자가 AI 추천을 실제로 따를까요? 저항은 없나요?압박

매우 중요한 지적입니다. 현장 저항은 모든 AI 도입의 공통 과제입니다.

저희의 대응:
  • 강제가 아닌 추천: Director App은 적용/무시 선택권을 작업자에게 부여. 최종 판단은 사람이 합니다.
  • 투명한 근거: "왜" 이 추천이 나왔는지 유사 코일 정보와 확률을 함께 제시. 블랙박스가 아닙니다.
  • 점진적 도입: 2026 H2에 교육부터 시작, 강제 적용이 아닌 참고 도구로 시작합니다.
  • 결과 추적: 추천 적용 결과가 자동 기록되므로, 실제로 효과가 있다는 것을 데이터로 보여줄 수 있습니다.
